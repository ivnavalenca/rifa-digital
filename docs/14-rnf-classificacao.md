# Classificação de Requisitos Não Funcionais

Este documento organiza os requisitos não funcionais por categoria.

------------------------------------------------------------------------

## Segurança

-   impedir seleção duplicada de números
-   proteger dados pessoais dos usuários
-   registrar logs das operações

------------------------------------------------------------------------

## Performance

-   tempo de resposta inferior a 2 segundos
-   capacidade de múltiplos acessos simultâneos

------------------------------------------------------------------------

## Usabilidade

-   interface simples e intuitiva
-   acesso por dispositivos móveis

------------------------------------------------------------------------

## Confiabilidade

-   garantir integridade dos dados
-   manter histórico das transações

------------------------------------------------------------------------

## Manutenibilidade

-   código modular
-   facilidade de atualização do sistema
